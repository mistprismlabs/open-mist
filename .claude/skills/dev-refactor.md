# /dev-refactor 代码重构

用户指定重构目标，执行安全重构流程：

1. 分析现状：识别代码坏味道、理解现有行为
2. 建立安全网：确认现有测试覆盖，必要时补充
3. 小步重构：每步只做一件事，保持行为不变
4. 验证：每步后运行测试确认行为一致
5. 提交、推送、部署

约束：
- 重构不改变外部行为（纯内部结构改善）
- 每个 commit 只做一件事（便于 revert）
- 优先使用 IDE 级重构：重命名、提取方法、移动文件
- 不要一次改太多，宁可多步小改
